### ADR-1: Single-File Bundle Strategy

**Decision:** Use esbuild with `--bundle --format=esm` to produce a
single `dist/index.js` file with all internal modules inlined.

**Location:** Build configuration (package.json scripts)

**Rationale:** Container deployment copies one file. No path issues.
External dependencies (`@opencode-ai/plugin`, `zod`) are provided by
the runtime and must be marked `--external`.

**Alternatives Considered:**
- Alternative A: Multiple files with `tsc` output.
  *Rejected:* Relative imports break in container ESM resolution.
- Alternative B: Webpack with code splitting.
  *Rejected:* Async chunks not supported by opencode plugin loader.
- Alternative C: Rollup with tree-shaking.
  *Rejected:* More complex config for same result as esbuild.

**Cost of Reversal:** MEDIUM — switching bundlers requires updating
build scripts, CI/CD, and testing all imports resolve correctly.

### ADR-2: Hook-Based Identity Injection (SCAN+REPLACE)

**Decision:** Use `experimental.chat.system.transform` hook with
SCAN+REPLACE logic to inject identity on every transform event.

**Rationale:** The transform hook fires on every compaction, ensuring
identity is re-injected exactly when it would be lost. SCAN+REPLACE
is idempotent — no duplicate blocks accumulate.

**Alternatives Considered:**
- Alternative A: Static system prompt in agent config.
  *Rejected:* Gets overwritten by platform on compaction.
- Alternative B: messages.transform for identity injection.
  *Rejected:* Noisier; identity in user message is confusing.
- Alternative C: Prepend on every tool.execute.before event.
  *Rejected:* Wrong lifecycle; does not fire on compaction.

**Cost of Reversal:** LOW — change the hook implementation. Logic is
isolated to the hooks file.

### ADR-3: Sequential Layer Pipeline

**Decision:** Mode pipelines execute layers sequentially.

**Rationale:** Dependencies are linear (Layer N+1 depends on Layer N).
Sequential execution gives deterministic ordering and clear traces.

**Alternatives Considered:**
- Alternative A: Parallel execution with Promise.all.
  *Rejected:* Requires mock inputs for dependent layers.
- Alternative B: Event-driven pipeline.
  *Rejected:* Harder to debug; unclear execution order.

**Cost of Reversal:** MEDIUM — requires restructuring mode pipeline
and adding mock input generation.

### ADR-4: DiscoveryResult as Single Source of Truth

**Decision:** All mode tools receive a unified DiscoveryResult.

**Rationale:** Single scan, shared result, consistent data.

**Alternatives Considered:**
- Alternative A: Each tool scans independently.
  *Rejected:* Redundant I/O, inconsistent data.
- Alternative B: Pre-computed intelligence in files.
  *Rejected:* Files go stale between invocations.

**Cost of Reversal:** LOW — change the discovery call to per-tool.

### ADR-5: Validation as Warning, Not Error

**Decision:** Validation returns warnings; pipeline always advances.

**Rationale:** Missing headings should not block the pipeline.

**Alternatives Considered:**
- Alternative A: Hard fail on missing sections.
  *Rejected:* Too brittle; blocks forward progress.
- Alternative B: Silent ignore.
  *Rejected:* No feedback loop for improvement.

**Cost of Reversal:** LOW — change validateLayerContent to throw.
